  • 需要把字符映射到数组也就是哈希表的索引下标上，因为字符a到字符z的ASCII是26个连续的数值，所以字符a映射为下标0，相应的字符z映射为下标25

  • 同样在检查字符串t中是否出现了这些字符时，对t中出现的字符映射哈希表索引上的数值再做-1的操作

  • 最后检查一下，record数组如果有的元素不为零0，说明字符串s和t一定是谁多了字符或者谁少了字符

  • 刚开始的时候，我打算使用Java的集合类来模拟哈希表解决问题的，但并没有发现太合适的类，HashSet和HashMap好像都不太符合要求，并且我对于这些类的相关操作也有些模糊了，之后有时间我会对Java中常用的类进行一个总结，希望对你有所帮助！

LeetCode.142.环形链表2

发表于 2023-11-12 更新于 2024-01-16

题目描述

我的题解

我的思考

  • 这道题不仅考察对链表的操作，还需要一定的数学运算

  • 我们首先要判断这条链表是否有环：

    • 我们设置快慢指针，如果有环，那么fast指针一定是先进入环的，通过判断fast指针是否为空或者fast.next是否为空来判断是否有环

  • 如果确实有环，我们需要判断环的入口在哪：

    • 首先第一点，如果有环的话，那么快慢指针一定是在环中相遇的，这是毋庸置疑的
    • 其次，我们需要判断环的入口在哪里了
    • 假设头节点到环的入口需要x个节点，环的入口距离快慢指针相遇位置有y个节点，从相遇节点到环的入口有z个节点
    • 相遇时，slow指针已经走了x + y个节点，fast指针已经走了x + y + n（y + z）个节点，n表示fast指针在环内走的圈数
    • 因为fast指针走过的节点树是slow指针的两倍，可以得到x + y + n（y + z）= 2（x + y）
    • 化简得到x = （n - 1）（y + z） + z，这里的n一定是大于1的，因为fast至少需要走一圈才能追上slow指针
    • 这个公式说明，头节点到环入口需要的节点个数是n - 1圈加上从相遇节点到环入口的距离
    • 那么我们重新设置一个从头节点开始和相遇节点每次同时移动一个节点，那么它们最终会相遇在环入口节点位置

  • 可能有同学要问：为什么第一次在环中相遇，slow指针走了x + y个节点，而不是x + 若干环长度 + y呢？

    • 我们要知道，在slow指针第一次进入环时，fast指针最多和slow指针相差n - 1个节点，n为环中节点数
    • 我们假设slow指针走了一个环的距离，即n个节点，那么fast指针走了2n个节点
    • 然而，2n > n - 1 + n，说明在slow完整走一个环之前，fast就已经追上slow指针了

我的思考

  • 移出链表中元素通常有两种情况：

    • 移出头节点
    • 移出中间节点

  • 对于移出中间节点，因为是单链表，所以我们需要找到待移出节点的前一个节点来继续移出，而对于移出头节点来说，我们有两种思路来进行移出：

    • 直接使用原来的链表进行移出
    • 设置一个虚拟头节点来进行移出

  • 我的第一种写法就是直接使用原来的链表进行移出头节点，通过head = head.next将头节点后移实现移出头节点，因此我需要将处理头节点和处理中间节点分开进行，但如果我们设置一个虚拟头节点，那么就可以将头节点转换为中间节点，头节点就可以和中间节点一起处理了！
